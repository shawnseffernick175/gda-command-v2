# GDA Command Tool — Ownership Model (Corrected)

## The Truth

**This is Envision's tool.** Built for Shawn, operated by Envision (OU-I, Defense & Mission Systems).

Riverstone Solutions and PD Systems are **teaming partners**, not users. They will never log in. They will never share Envision's pipeline. They will never see Envision's financials.

GDA-corporate is the **branding/holding context** — the story Shawn tells the customer and the doctrine he answers to. It is not a shared platform.

## What changes

| Before (wrong) | After (correct) |
|---|---|
| Tool = GDA enterprise platform with 3 OUs as co-equal tenants | Tool = Envision workspace; partners are intel records |
| OU tag on every record (Envision/Riverstone/PD-SYS) | Default = Envision. Partner tag added only when teaming is in play |
| Financials roll up GDA-wide | Financials are Envision-only. GDA-rollup is a story for proposals |
| Pipeline shared across OUs | Pipeline is Envision's. Partner work appears only when relevant to a teaming decision |
| Doctrine enforces all 3 OUs | Doctrine is Envision's operating discipline; the corporate vision is context |

## The new role of Riverstone + PD Systems data

They become **Partner Intel Profiles** — read-only cards Envision references to answer two questions:

1. **What are they pursuing?** → Avoid stepping on the same opportunity unless we're teaming.
2. **What do they bring?** → When Envision needs HUBZone (Riverstone) or training capacity (PD Systems), pull them onto the proposal.

That's it. They're not co-tenants. They're partner intel.

## What the Doctrine still demands

The CEO's doctrine still applies — but as **Envision's operating standard**, because Envision lives inside GDA and Shawn must align. The 7 principles (Alignment, Ethics, Teamwork, Data First, Relentless Execution, Relationships, Market/Mission/Brand Focus) govern HOW Envision operates inside GDA, not how the tool federates across OUs.
